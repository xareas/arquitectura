# .Domain

## Capa de Dominio

:::tip Tip
  En el desarrollo de software, la capa de dominio es crucial para representar las reglas de negocio de manera clara. El patrón Repositorio es una parte esencial de esta capa, permitiéndonos separar la lógica de acceso a datos de la lógica del dominio. Con interfaces de Repositorio, podemos definir operaciones CRUD para nuestras entidades sin preocuparnos por la fuente de datos.
:::

* Entidades
* Value Objects (DTO)
* Domain Services (Managers)
* Domain Events
* Interfaces de Repositorio

### Entity
Una Entidad es un objeto con sus propias propiedades.(estado, datos) y métodos que implementa el negocio.
Una entidad es representado por su identificador único (Id). Dos entidades con diferentes ID se consideran entidades diferentes.

### Value Object
Un objeto de valor es otro tipo de objeto de dominio. Se identifica por sus propiedades en lugar de por una
Identificación única. Eso significa dos objetos de valor con las mismas propiedades se consideran el mismo objeto.
Los objetos generalmente se implementan como inmutables y en su mayoría son mucho más simples que las Entidades.

### Aggregate & Aggregate Root
Agregado y raíz agregada: un agregado es un clúster de objetos (entidades y objetos de valor) unidos por
un objeto Raíz. La raíz agregada es una tipo específico de entidad con algunas responsabilidades adicionales.

### Repository( Interface )
Un repositorio es una colección de acciones es decir una interfaz  que se utiliza en las capas de dominio y la aplicación
para acceder al sistema de persistencia de datos (base de datos). Oculta la complejidad del DBMS. La capa de dominio contiene las interfaces de los repositorios.

### Domain Services
Un servicio de dominio es un servicio sin estado que implementa las reglas de negocio centrales del dominio. Es
útil para implementar lógica de dominio que depende de tipo de agregado múltiple (entidad) o algún tipo externo
servicios.

### Specification
Una especificación se utiliza para definir nombres,filtros reutilizables y combinables para entidades y otros
objetos de negocio. En c# el uso de predicados es un tipo de especificacion.

### Domain Event
Un Evento de Dominio es una forma de informar otros servicios de manera débilmente acoplada, cuando un
ocurre un evento específico del dominio. Estos tipos de eventos, van relacionados especificamente a eventos que
normalmente ocurren dentro del un contexto limitado(Bounded Context,Modulo,Microservicio). Existe otro tipos de eventos
que son los Integration Events, para eventos que estan fuera del modulo o microservicios, o del contexto.
