# .Infraestructure.EfCore

En una implementación DDD, es posible que tenga una unico proyecto de infraestructura
para implementar todas las abstracciones e integraciones, o es posible que tenga diferentes proyectos para cada dependencia.

Sugerimos un enfoque equilibrado; Crear proyectos separados para principales dependencias de infraestructura (como Entity Framework Core)
y un proyecto de infraestructura común para otras infraestructuras.

:::tip Infraestructura
Es probable que tenga un solo proyecto de infraestuctura, pero no siempre sera asi, debe encontrar un balance.
:::

El startup de solución de Makers tiene dos proyectos uno para el Manejo de Persitencia con Entity Framework y otro donde residen
todo lo de la componentes de infraestructura. Como decimos debe encontrar un balance.
